CSC 120 Programming Assignment 2

Note: You should not work on this assignment during lecture or lab class-time unless:
  1. You have completed all of the in-class assigned work for the day, and
  2. You have permission from the instructor.

This assignment must be handed in to the PA2 dropbox in D2L before the beginning of class on the due date.

Overview:

You are to create your own Java class that contains a minimum of four (4) properties, at least two of which must be some numeric type (Integer or Double).  Then you must write code in an MUPanel class that declares and instantiates at least five (5) objects from your class and displays lines of text that shows that the objects from your class work correctly.

Instructions:

  1. Initial Set-up.

    1. Create a NetBeans Java Application named PA2, and copy the MUPanel.java file from your Lab4 project into the src folder of the PA2 project.

    2. Do not try to run the program (yet). The program from Lab4 uses ClothingItem and Coffee objects.  However, you do not have those two classes in the src folder of your project, so NetBeans will complain about the fact that the those classes doesn't exist!

  2. Create your class.

    1. You need to design your class before you continue! You are not allowed to use a ClothingItem class or a Coffee class or a BaseballHitter class or a TVSet class for this assignment.  Try to think of a class that nobody else will use.

    2. In the Files tab of your project, right-click on the src folder and choose "New | Java Class..." and set the Class Name: to be the name you've chosen for your class. Be sure you capitalize the name and don't use any spaces. For example you could use Airplane, but not airplane, and not Air Plane.

    3. Edit the new class. Change the comment header to say the name of the class, your name, and the date.

    4. Your class will need at least 4 properties (private data members), and at least two of them must be a numeric type (you can use either Integer or Double as the data type for these properties).

    5. Make a constructor for your class that accepts a parameter for each of the properties, and uses them to initialize the corresponding properties of the object.

    6. Make "getter" methods for each of the properties of your class.  The "getter" methods should return the proper data type for each of the properties.

    7. Make a toString method that returns a String that describes the object in terms of its properties.  Refer to the toString method in the classes from Lab4.

    8. Make a method that performs a calculation with two or more numeric properties of your class, similar to the calculateTotalSpent method of the ClothingItem class and the calculatePricePerOunce method of the Coffee class.

    9. You are not required to have "setter" methods for the properties of your class for this assignment, but you may want to make them for your class for completeness.

  3. Modify the MUPanel to use your class.

    1. The MUPanel from Lab4 declares ClothingItem and Coffee objects.  Change the declarations in the MUPanel.java file so that you are declaring at least five (5) objects from your class.

    2. Instantiate your objects so that the constructor method has the correct number and type of values being passed into the constructor between the parentheses.  Once again, refer to the way this was done in Lab4.

    3. The System.out.println or muta.println statements used in Lab4 must be modified to use your objects and to print messages that make sense for your class.  For instance, if your class is Airplane, change the data printed with the println statements so that the output makes sense for Airplane objects.

    4. Use the numeric properties of your objects to perform some kind of calculation.  For instance, if you had a BaseballPlayer class, you could add up the hits for all players who play for the same team and print the total.  Use the "getter" methods of your class to retrieve data from the numeric properties so that those values can be used in a calculation.  Print the calculated value using System.out.println or muta.println, whichever you prefer.

    5. Test your program, and fix & retest as needed.

Turning in your Program for Grading:

When you have completed your NetBeans project for PA2 (including the comment header at the top of the two files with your name and having your name appear in the title of the frame window, etc.), use File Explorer to make a .zip file of the PA2 folder in your H:\CSC120\Java folder so that the file is named PA2.zip.  Next, run your program and obtain a screen capture that shows the output generated by your program.  Then upload those two files to the PA2 dropbox in D2L.
